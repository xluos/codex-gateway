# Security Policy

## 支持范围

当前安全修复默认以最新主分支为准。历史提交或长期未同步的私有分支不承诺提供补丁。

## 报告方式

如果你发现了安全问题，请不要直接公开提交 issue、讨论贴或包含漏洞细节的 PR。

目前仓库没有单独的安全邮箱。请通过仓库维护者提供的私下联系方式报告问题；如果托管平台支持私密安全报告，也优先使用该渠道。

在报告时，建议包含：

- 漏洞类型与影响范围
- 复现步骤或最小 PoC
- 可能受影响的配置条件
- 你认为的修复方向或缓解措施

## 处理原则

- 先确认问题是否真实存在及影响范围
- 在可行情况下先准备修复，再公开披露细节
- 修复发布后再补充必要的公开说明

## 敏感信息处理

请不要在公开渠道中披露以下内容：

- 本地 `config.yaml`
- OAuth 凭证文件
- 实际 API Key
- 含有真实 prompt、日志或请求体的调试输出

如果你需要提供日志，请先做脱敏处理。
