# Contributing

感谢你关注 Codex Gateway。

这个项目当前以本地单用户场景为核心，欢迎改进稳定性、兼容性、文档质量和开发体验，但不鼓励把仓库推进到多租户、计费系统或重型平台化方向。

## 提交问题

- 在提交 issue 之前，请先确认问题可以在当前主分支复现。
- 提供最小复现步骤、实际结果、期望结果，以及必要的运行环境信息。
- 如果问题与认证、配置或日志有关，请去除 API Key、OAuth 凭证、请求内容和其他敏感信息后再提交。

## 提交代码

- 优先提交小而聚焦的改动，避免把无关重构和功能修改混在一起。
- 行为变更应同时更新 README、示例配置或相关文档。
- 如果改动影响现有命令、接口或配置项，请补充最小验证说明。
- 新增依赖前请先说明必要性，保持项目轻量。

## 提交前检查

- 运行 `go test ./...`
- 确认文档中的命令、路径和端口与当前实现一致
- 确认未提交本地配置、运行日志、OAuth 凭证或其他私有文件

## Pull Request 说明

- 清楚说明改动目的、范围和兼容性影响。
- 如果改动涉及接口行为，请附上示例请求或输出片段。
- 如果改动尚未覆盖自动化测试，请明确列出手工验证方式和未覆盖风险。

## 讨论边界

更容易被接受的方向：

- OpenAI 兼容性修正
- OAuth 与 API Key 模式的稳定性修复
- CLI 可用性改进
- 安装、排障和文档完善

通常不会优先接受的方向：

- 多用户、多租户能力
- 计费、配额、管理后台
- 与项目定位不一致的重平台化改造
